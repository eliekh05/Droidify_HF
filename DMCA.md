# DMCA and Takedown Policy

## What Droidify actually does

Droidify reads publicly available metadata — device names, codenames, ROM names, version numbers, and download links — from public APIs and web pages run by the original projects. That is the entire scope of what it does. It does not host files. It does not store, redistribute, mirror, copy, or touch any firmware, ROM, recovery image, or binary at any point.

Every download link on Droidify points directly to the servers of whoever made the file. LineageOS mirrors. OrangeFox CDN. TWRP's own servers. GitHub Releases. SourceForge. The files go from their servers to your device. Droidify is not in that path anywhere.

This is what search engines do. This is what XDA does when it links to a download. Displaying public metadata and linking to the original source is not redistribution. It has never been redistribution.

---

## We built this because the gap was real

Since Android 1.0 there has been no single place to find what ROMs and tools exist for a given device. All the data was publicly available the entire time. Nobody built the aggregator. Droidify is not scraping private data or bypassing anything — it is reading the same public APIs that any browser would read when visiting those sites.

Your project appears on Droidify because you published it publicly. We are pointing people toward it. That is the same thing you did when you put it on GitHub or XDA.

---

## If you want your project removed

No lawyer. No formal filing. Open a GitHub issue with the `takedown-request` label, tell us the project name and the URL on Droidify where it appears, and we will remove it within 48 hours. No questions asked. We have no interest in indexing anything from someone who does not want to be there.

If you want it permanently blocked so it never reappears even if a scraper picks it up again later, say so and we will add it to a blocklist.

---

## Formal DMCA notices

If you genuinely believe Droidify is hosting or redistributing a file you own the copyright to — it is not, but if you believe otherwise — open a GitHub issue and we will respond within 48 hours.

A valid DMCA notice needs to identify the specific copyrighted work and the exact URL on Droidify where you believe it appears. A request that says "remove my project" without a specific copyright claim is a standard takedown request — use that template, it is faster.
